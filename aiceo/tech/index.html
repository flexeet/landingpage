Decision Unit Structure

{ "domain": "product | operations | hr | marketing", "type": "principle | boundary | choice | trade_off | failure_scar", // Core Decision Fields "default_behavior": "What owner USUALLY does", "context_trigger": "WHEN this applies", "kill_criteria": "STOP conditions", "what_owner_avoids": ["list", "of", "things"], // Confidence Signals "confidence_source": "experience | pattern | intuition", "risk_tolerance": "low | medium | high", "signal_strength": "high | medium | low", // Quote & Context "interpreted_quote": "Original quote Bahasa Indonesia", "speaker": "Owner Name (Business)" }
Types

5 Decision Types

Type Purpose Example
principle Core belief that guides behavior "Always prioritize function over aesthetics"
boundary Hard limits / red flags "Never use fleece, parasut, or knit"
choice Preference when options exist "Prefer Drop Shoulder for textured fabrics"
trade_off How to balance competing priorities "Loose Fit 60% vs Regular Fit for weather"
failure_scar Lessons from past mistakes "Lost money when ignored market fit"

Creative Task Rules

‚úÖ WAJIB dari Knowledge
